# Citation Validator for Academic Integrity
## Detecting AI-Hallucinated References in Scientific Literature

**Built in response to:** [Nature article (April 8, 2026) - "Hallucinated citations are polluting the scientific literature"](https://www.nature.com/articles/d41586-026-00969-z)

### 🎯 The Problem

Nature recently reported that **2-6% of 2025 papers contain hallucinated citations** - fabricated references generated by AI tools. These "Frankenstein citations" mix real authors, titles, and journals into non-existent papers, polluting the scientific record.

**Our response:** A free, open-source tool built in ~3 hours that matches Nature's findings on our test data (5% invalid citation rate).

### 🔬 How It Works

**Three-Tier Validation Pipeline:**

1. **CrossRef API** - Validates DOIs and retrieves verified metadata
2. **OpenAlex API** - Fallback search for non-DOI citations by title/author  
3. **Groq AI** (Llama 3.1) - Detects Frankenstein citations using contextual analysis

**Enhanced Heuristics** (NEW):
- Generic pattern detection (vague titles, placeholder authors)
- Temporal consistency checks (future years, anachronisms)
- Metadata similarity scoring (Jaccard similarity)
- Missing field detection
- Inconsistent venue data flagging

### 📄 How to Get BibTeX Citations for Testing

**Option 1: Google Scholar (Easiest for Quick Tests!)**
Perfect for testing specific papers or building a sample .bib file:
1. Search for any paper on [Google Scholar](https://scholar.google.com)
2. Click the "Cite" button (quotation mark icon)
3. Click "BibTeX" at the bottom
4. Copy the BibTeX entry
5. Paste into the web app or save as a `.bib` file

**Example:** Try searching "Nature hallucinated citations 2026" to test the very article that inspired this tool!

**Option 2: Your LaTeX Bibliography**
If you're writing in LaTeX, you already have a `.bib` file:
```bash
python3 citation_validator.py yourpaper/bibliography.bib
```

**Option 3: Reference Manager Export**
Export from Zotero, Mendeley, EndNote, JabRef:
- File → Export → BibTeX format
- Save as `.bib` file
- Upload to web app or run via command line

**Option 4: Manual BibTeX Entry**
Create entries by hand for specific citations you want to verify:
```bibtex
@article{AuthorYear,
  title={Paper Title},
  author={Last, First and Last2, First2},
  journal={Journal Name},
  year={2025},
  volume={123},
  pages={45-67},
  doi={10.1234/example}
}
```

### ⚙️ How It Works (Technical Details)

**Validation Pipeline:**

1. **Parse BibTeX** → Extract entries with regex
2. **DOI Validation** → Query CrossRef API
3. **Metadata Comparison** → Title/year/author matching
4. **OpenAlex Fallback** → Search for non-DOI citations
5. **Heuristic Analysis** → Pattern detection
6. **AI Assessment** → Groq Llama 3.1 analysis (optional)

**APIs Used (all free):**
- **CrossRef**: 50 requests/second, no authentication
- **OpenAlex**: No rate limit with polite header
- **Groq**: 30 requests/minute, 7000/day (free tier)
